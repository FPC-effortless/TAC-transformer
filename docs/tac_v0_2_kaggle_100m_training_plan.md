# TAC v0.2 Kaggle 100M+ Training Plan

Short answer:

> Yes, TAC can attempt a 100M+ parameter pilot on Kaggle, but it should be
> treated as a feasibility/profile run first, not a full pretraining claim.

Kaggle provides notebook accelerators with quota limits. The relevant practical
constraint is not whether a 100M model can be instantiated; it is whether the
run can train long enough, checkpoint reliably, resume cleanly, and produce
matched controls.

Useful public reference:

- Kaggle Notebooks documentation: https://www.kaggle.com/docs/notebooks
- Kaggle GPU usage guidance: https://www.kaggle.com/docs/efficient-gpu-usage

## Recommended Pilot Shape

Generated by:

```bash
python experiments/estimate_tac_v02_100m_kaggle.py
```

Recommended first shape:

| Field | Value |
|---|---:|

| vocab_size | `8192` |
| d_model | `512` |
| n_heads | `8` |
| n_layers | `8` |
| n_programs | `24` |
| seq_len | `512` |
| attention_window_size | `128` |

This is the smallest clean 100M+ profile found by the current estimator that
keeps the Kaggle fast TAC memory path.

## Pilot Command

Run this first on Kaggle:

```python
!python kaggle/train_best_tac_agentic.py \
  --preset kaggle_fast_tac \
  --scale smoke \
  --vocab-size 8192 \
  --d-model 512 \
  --n-heads 8 \
  --n-layers 8 \
  --n-programs 24 \
  --seq-len 512 \
  --attention-window-size 128 \
  --steps 1000 \
  --batch-size 1 \
  --grad-accum-steps 32 \
  --eval-every 250 \
  --eval-batches 2 \
  --checkpoint-every 250 \
  --precision fp16 \
  --device auto \
  --max-seconds 21600 \
  --stop-buffer-seconds 1200 \
  --skip-end-specialization-on-time-stop \
  --output-dir /kaggle/working/tac_v02_100m_pilot
```

## Pilot Success Criteria

The pilot passes only if:

- the model initializes on Kaggle GPU;
- training completes at least one checkpoint;
- `metrics.jsonl` is written;
- `last.pt` is written;
- loss is finite;
- gradient health is nonzero;
- throughput is measured;
- resume from `last.pt` works.

The pilot does not validate TAC v0.2.

## Required Control Runs

Only after the pilot is healthy:

1. TAC carried-state run.
2. TAC reset-state control.
3. Parameter-matched vanilla control.
4. Program knockout evaluation.
5. Compression evaluation.
6. Real-repository workflow evaluation.

## Expected Risks

- Kaggle GPU memory may require lower `batch-size` or shorter `seq-len`.
- Quota limits may require multiple resumed sessions.
- A 100M model can train but still fail the mechanism-survival gates.
- The current corpus may be too small for meaningful scale evidence.
- The first bottleneck may be data quality rather than architecture.

## Decision

Kaggle is suitable for a Stage-4 pilot. It is not suitable for making a strong
foundation-model-scale claim. Treat it as a low-cost survival test before using
larger compute.
